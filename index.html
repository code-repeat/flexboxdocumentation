Flex Container

The Flex Container is a tag that involves the flex items, to indicate display: flex, this tag passes to a Flex Container.

1 display

This defines a flex container; inline or block depending on the given value. It enables a flex context for all its direct children.

See the Pen rNOoNaW by Code Repeat (@code-repeat) on CodePen.

2 flex-direction

The flex-direction property defines in which direction the container wants to stack the flex items. By default it is strong row, so when the display: flex; is added, the elements are in line, side by side.

The change from row to column usually happens when we are defining styles in media queries for mobile. This ensures that the content is presented in a single column.

See the Pen LYpMYgg by Code Repeat (@code-repeat) on CodePen.

3 flex-wrap

Defines whether the items should break the line or not. By default they do not break lines, this causes the flex items to be compressed beyond the content limit.

This is usually a property that is almost always defined as flex-wrap: wrap; So when one of the flex items reaches the content limit, the last item moves to the column underneath and so on.

See the Pen flex-wrap by Code Repeat (@code-repeat) on CodePen.

4 flex-flow

Flex-flow is a shortcut to the flex-direction and flex-wrap properties. You will not see much of its use, because generally when we change the flex-direction column, we keep the flex-wrap pattern which is nowrap.

E quando mudamos o flex-wrap para wrap, mantemos o padrão do flex-direction que é row.

5 justify-content

Align flex items in the container according to direction. The property only works if the current items do not occupy the entire container. This means that when defining flex: 1; or something similar in the items, the property will no longer have a function

Excellent property to be used in cases where you want to align an item on the left and another on the right, as in a simple header with branding and navigation.

See the Pen justify-content by Code Repeat (@code-repeat) on CodePen.

6 align-items

The align-items aligns the flex items according to the container axis. The alignment is different for when items are in columns or rows.

This property allows the long-awaited central alignment on the vertical axis, something that was previously only possible with different hacks.

See the Pen MWaLJEK by Code Repeat (@code-repeat) on CodePen.

7 align-content

Aligns the container lines to the vertical axis. The property only works if there is more than one line of flex-items. For this, the flex-wrap needs to be wrap.

Furthermore, its effect will only be seen if the container is larger than the sum of the lines of the items. This means that if you do not define height for the container, the property has no influence on the layout.

See the Pen OJydWBY by Code Repeat (@code-repeat) on CodePen.

Flex Item

Flex Items are the direct children of the Flex Container, remembering that a tag becomes a flex container from the moment you define display: flex. It is possible that a Flex Item is also a Flex Container, just define display: flex in it. So the children of that item will also be flex items.

1 flex-grow

Sets the ability of a flex item to grow. By default the value is zero, so the flex items occupy a maximum size related to their internal content or to the defined width.

When setting 1 for all Flex Items, they will try to have the same width and will occupy 100% of the container. I mean try because if an element has a very wide content, it will respect the same.

If you have a line with four items, where three are flex-grow: 1 and a flex-grow: 2, flex-grow: 2 will try to occupy 2 times more extra space than the other elements.

NOTE: justify-content does not work on items with defined flex-grow.

See the Pen yLYZMyM by Code Repeat (@code-repeat) on CodePen.

2 flex-basis

Indicates the initial size of the flex item before distributing the remaining space.

When we define flex-grow: 1; and we have auto on basis, the remaining amount to occupy the container is distributed around the content of the flex-item.

See the Pen rNOPywd by Code Repeat (@code-repeat) on CodePen.

3 flex-shrink

Sets the item's size reduction capability.

See the Pen qBOgrVg by Code Repeat (@code-repeat) on CodePen.

4 flex

Shortcut to the flex-grow, flex-shrink and flex-basis properties. You will usually see the flex property in the flex items instead of each of the separate values.

For better consistency between browsers, it is recommended to use the flex property instead of each separate property.

In the example it is possible to see the same configurations as the flex-basis example, but now using only the flex property.

See the Pen qBOwXdP by Code Repeat (@code-repeat) on CodePen.

5 order

Modifies the order of flex items. Always from lowest to highest, order: 1 appears in front of order: 5.

See the Pen LYpvjZe by Code Repeat (@code-repeat) on CodePen.

6 align-self

Align-self is used to define the specific alignment of a single flex item within our container. If a value is assigned, it will go over what is assigned in the container's align-items.

See the Pen eYpoEvB by Code Repeat (@code-repeat) on CodePen.
